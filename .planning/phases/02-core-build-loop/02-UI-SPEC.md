# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — vanilla HTML/CSS |
| Icon library | none — text labels only for v1 |
| Font | system-ui, sans-serif (OS default) |

**Rationale (source: RESEARCH.md + CLAUDE.md):** The project is a pure Three.js vanilla JS application — no React, no Tailwind, no shadcn. UI elements (piece tray, instruction panel) are HTML `<div>` overlays positioned absolutely over the `<canvas>` via plain CSS. No component library is needed or appropriate for this architecture.

## Component Inventory

### 1. Piece Tray (`#tray`)

**Position:** Fixed bottom bar, full width, above canvas. Height: 72px.

**Structure:**
```
#tray
  .tray-items          ← flex row, centered, gap: 8px
    .tray-item         ← 44px x 44px click target
      .tray-swatch     ← 40px x 40px colored square, border-radius: 4px
    .tray-item.selected ← adds accent border ring + background tint
```

**Visual states:**
- Default: swatch shows piece color, `rgba(255,255,255,0.12)` 1px border
- Selected (`heldPiece` set): 2px `#e94560` border ring, `rgba(233,69,96,0.15)` background, no scale transform
- Hover: `rgba(255,255,255,0.20)` border, cursor: pointer
- Empty tray (all pieces placed in step): tray bar remains visible, `.tray-items` shows no children

**CSS approach:** Position `fixed; bottom: 0; left: 0; right: 0;` over the canvas. `z-index: 10`.

---

### 2. Instruction Panel / HUD (`#hud`)

**Position:** Fixed top-left corner. Width: 220px. Padding: 16px 24px.

**Structure:**
```
#hud
  .hud-step-counter    ← "Step 3 of 7" — Heading size, 600 weight, #e8eaf6
  .hud-step-desc       ← step description text — Body size, 400 weight, secondary text color
  .hud-progress        ← thin progress bar (optional visual, 4px height, accent color fill)
```

**Visual states:**
- Default: panel visible, step counter and description rendered
- Step advance: counter and description update immediately (no animation — instant text swap)
- Build complete (all steps done): panel shows "Build complete!" — no HUD update after that (completion screen is Phase 3)

**CSS approach:** Position `fixed; top: 16px; left: 16px;` over the canvas. `z-index: 10`. Background `rgba(22,33,62,0.88)`, border-radius: 8px.

---

### 3. Ghost Overlay Mesh (Three.js — not HTML)

Not a CSS component. Visual contract only:

- Material: `MeshStandardMaterial`, `transparent: true`, `opacity: 0.35`, `depthWrite: false`, `side: THREE.DoubleSide`
- Color: same hex as the target piece's color from step JSON
- `renderOrder: 1` (renders after opaque geometry)
- Polygon offset fallback (if z-fighting): `polygonOffsetFactor: -1, polygonOffsetUnits: -1`
- Source: RESEARCH.md Pattern 4, CLAUDE.md explicit pattern

---

### 4. Placement Feedback Flash (GSAP — not HTML)

Not a CSS component. Visual contract only:

- Success: placed brick material tweens from `rgb(76,175,80)` → brick's own color over 400ms, ease: `"power2.out"`
- Rejection: ghost material tweens from its piece color → `rgb(229,57,53)` → back to piece color, 300ms total (150ms out, 150ms in), ease: `"power1.inOut"`
- Source: RESEARCH.md Pattern 6

## Interaction Contract

### Mouse Interaction Model (source: CLAUDE.md, REQUIREMENTS.md)

| Action | Trigger | Behavior |
|--------|---------|----------|
| Orbit camera | Left-click drag (pointer delta > 5px) | OrbitControls handles; no raycast fires |
| Zoom camera | Mouse wheel | OrbitControls handles |
| Pan camera | Right-click drag | OrbitControls handles |
| Pick piece from tray | Left-click on `.tray-item` | DOM click; `holdPiece(id)` in state; tray re-renders with selected class; ghost appears at step target |
| Confirm placement | Left-click on ghost mesh (pointer delta < 5px) | Raycast hits ghost mesh; validates against step target; success or rejection feedback fires |
| Invalid placement | Left-click on baseplate or brick (not ghost), pointer delta < 5px | Rejection flash on ghost; no state change |

**Disambiguation threshold:** 5px pointer delta (source: RESEARCH.md Pattern 2). If orbit sensitivity issues arise, increase to 8px.

**Cursor states:**
- Default (no piece held): `cursor: default` on canvas
- Piece held (hovering canvas): `cursor: crosshair` on canvas
- Hovering tray item: `cursor: pointer`

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Pick a piece" (tray item title attribute — no visible label button in Phase 2) |
| Step counter | "Step {N} of {M}" |
| Step description | Sourced from set JSON `step.description` field |
| Empty tray state | "All pieces placed — step complete!" (shown in `.hud-step-desc` momentarily before step advances) |
| Build complete (last step) | "Build complete!" (shown in HUD; full completion screen is Phase 3) |
| Invalid placement | No text copy — rejection is visual only (red flash on ghost); no toast or error text in Phase 2 |
| Loading state | "Loading set..." (shown in `#hud` `.hud-step-counter` during async fetch before set data resolves) |
| Destructive confirmation | Not applicable — no destructive actions in Phase 2 |

## States and Transitions

| UI State | Tray | HUD | Ghost |
|----------|------|-----|-------|
| Loading (fetch in flight) | Hidden | "Loading set..." | Hidden |
| Idle, no piece held | Shows step pieces, none selected | Shows "Step N of M" + description | Visible (always shows target) |
| Piece held | Shows step pieces, one selected | Unchanged | Visible (same ghost — click to confirm) |
| Success placement | Tray re-renders for new step | Counter and description update | Hides; new step ghost appears |
| Rejection (invalid click) | Unchanged | Unchanged | Red flash then returns to piece color |
| All steps complete | Tray empty | "Build complete!" | Hidden |

## Accessibility Contract

- All tray items have `title` attribute with piece type (e.g., `title="brick-2x4"`) for screen reader discovery
- HUD step counter has `aria-live="polite"` so step advances are announced
- Color is never the sole signal: selected tray item has border ring (shape) in addition to color change
- Ghost overlay is a 3D scene object — no ARIA needed
