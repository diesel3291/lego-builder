Create `frontend/src/state.js` — the single mutable state module for the entire build.

**Module-level private state:**
```javascript
let _setData = null;
let _currentStepIndex = 0;
let _heldPieceId = null;
const _placedCells = new Set();       // gridKey strings of all occupied cells across all steps
const _placedThisStep = new Set();    // piece IDs placed in the current step
```

**Imports:**
```javascript
import { gridKey, DIMS } from './grid.js';
```

**Exported functions (implement ALL of these):**

1. `loadSet(setData)` — Stores setData, resets `_currentStepIndex` to 0, clears `_heldPieceId`, clears `_placedCells`, clears `_placedThisStep`.

2. `getCurrentStep()` — Returns `_setData.steps[_currentStepIndex]` or `null` if `_setData` is null or build is complete.

3. `getTotalSteps()` — Returns `_setData.steps.length` or 0 if no set loaded.

4. `getCurrentStepNumber()` — Returns `_currentStepIndex + 1`.

5. `holdPiece(pieceId)` — Sets `_heldPieceId = pieceId`.

6. `releasePiece()` — Sets `_heldPieceId = null`.

7. `getHeldPieceId()` — Returns `_heldPieceId`.

8. `isOccupied(gridX, gridZ, layer)` — Returns `_placedCells.has(gridKey(gridX, gridZ, layer))`.

9. `placeBrick(piece)` — Takes a piece object `{id, type, gridX, gridZ, layer, ...}`. Uses `DIMS[piece.type]` to get `[cols, rows]`. Iterates `cx` from 0 to cols-1 and `rz` from 0 to rows-1, adding `gridKey(piece.gridX + cx, piece.gridZ + rz, piece.layer)` to `_placedCells`. Also adds `piece.id` to `_placedThisStep`. Returns nothing.

10. `getPlacedThisStep()` — Returns `_placedThisStep` (the Set).

11. `isStepComplete()` — Returns `true` if every piece in `getCurrentStep().pieces` has its `id` in `_placedThisStep`. Handles null current step by returning false.

12. `advanceStep()` — Increments `_currentStepIndex` (only if not at last step). Clears `_placedThisStep`. Sets `_heldPieceId = null`. Returns nothing.

13. `isBuildComplete()` — Returns `true` if `_currentStepIndex >= _setData.steps.length - 1` AND `isStepComplete()` is true. Returns `false` if no set loaded.

**Critical design rules:**
- No other module should mutate state directly — they call these exported functions
- `advanceStep()` must clear `_placedThisStep` and `_heldPieceId` (Pitfall 5 from RESEARCH.md)
- `placeBrick()` must mark ALL cells the piece occupies, not just the corner cell
- `getCurrentStep()` must guard against null `_setData` (Pitfall 7 from RESEARCH.md)

Create `frontend/src/ghost.js` — manages transparent ghost meshes showing where step pieces go.

**Imports:**
```javascript
import * as THREE from 'three';
import { getScene } from './scene.js';
import { getGeometry } from './geometry.js';
import { gridToWorld } from './grid.js';
```

**Module-level private state:**
```javascript
let _ghostMeshes = [];  // Array of THREE.Mesh — all currently visible ghosts
```

**Exported functions:**

1. `showGhost(piece)` — Creates a single ghost mesh for one piece object `{type, color, gridX, gridZ, layer, rotation}`.
   - Get geometry via `getGeometry(piece.type)` (do NOT clone — use the cached reference)
   - Create material: `new THREE.MeshStandardMaterial({ color: piece.color, transparent: true, opacity: 0.35, depthWrite: false, side: THREE.DoubleSide })`
   - Create mesh: `new THREE.Mesh(geometry, material)`
   - Set `mesh.renderOrder = 1`
   - Position via `gridToWorld(piece.gridX, piece.gridZ, piece.layer, piece.type)` and `mesh.position.copy(pos)`
   - Rotation: `mesh.rotation.y = THREE.MathUtils.degToRad(piece.rotation)`
   - Tag for identification: `mesh.userData.ghostPieceId = piece.id`
   - Add to scene: `getScene().add(mesh)`
   - Push to `_ghostMeshes` array
   - Return the mesh (needed by interaction.js for raycasting)

2. `hideGhost(pieceId)` — Removes the ghost mesh matching `pieceId` from scene.
   - Find mesh in `_ghostMeshes` where `mesh.userData.ghostPieceId === pieceId`
   - `getScene().remove(mesh)`
   - `mesh.material.dispose()` — ONLY dispose material, NEVER dispose geometry (Pitfall 3 from RESEARCH.md — geometry belongs to cache)
   - Remove from `_ghostMeshes` array

3. `showStepGhosts(step)` — Takes a step object `{pieces: [...]}`. Calls `hideAllGhosts()` first, then calls `showGhost(piece)` for each piece in `step.pieces`. Returns array of created meshes. This implements the "show all step ghosts simultaneously" design from RESEARCH.md Open Question 2.

4. `hideAllGhosts()` — Removes all ghost meshes from scene. For each mesh in `_ghostMeshes`: `getScene().remove(mesh)`, `mesh.material.dispose()`. Then sets `_ghostMeshes = []`.

5. `getGhostMeshes()` — Returns the `_ghostMeshes` array. Needed by interaction.js to include ghosts in raycaster targets.

**Critical design rules:**
- NEVER call `mesh.geometry.dispose()` — geometry is from the shared cache in geometry.js (Pitfall 3)
- ALWAYS call `mesh.material.dispose()` when removing a ghost (prevents material leak)
- Ghost opacity is exactly `0.35` (from UI-SPEC)
- `depthWrite: false` is mandatory to prevent z-fighting (from CLAUDE.md pattern)
- `renderOrder: 1` ensures ghost renders after opaque bricks
- `side: THREE.DoubleSide` so ghost is visible from any camera angle
