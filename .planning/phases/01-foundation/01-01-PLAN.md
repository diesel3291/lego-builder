Set JSON schema (v1) — to be created in sets/schema.md:
```
{
  "schemaVersion": 1,               // integer — bump on breaking changes
  "id": "string",                   // kebab-case, matches filename without .json
  "name": "string",                 // display name
  "description": "string",         // short description for set selection screen
  "pieceCount": integer,            // total pieces across all steps
  "steps": [
    {
      "stepNumber": integer,        // 1-indexed, sequential, no gaps
      "description": "string",     // human-readable instruction
      "pieces": [
        {
          "id": "string",           // unique within set, e.g. "step1-piece1"
          "type": "string",         // one of: brick-1x1, brick-1x2, brick-1x3,
                                    //   brick-1x4, brick-2x2, brick-2x3, brick-2x4,
                                    //   plate-1x1, plate-1x2, plate-1x4, plate-2x2,
                                    //   plate-2x4, slope-2x1, slope-2x2
          "color": "string",        // CSS hex, e.g. "#e3000b"
          "gridX": integer,         // stud column, can be negative
          "gridZ": integer,         // stud row, can be negative
          "layer": integer,         // 0 = on baseplate, +1 per brick height
          "rotation": integer       // Y-axis degrees: 0, 90, 180, or 270
        }
      ]
    }
  ]
}
```

  GET /api/sets/<id>  →  full set JSON object as above

**File: sets/schema.md**
Document the v1 set JSON schema (the data contract). This file is the authoritative
reference for all set authors and the Flask validator.

Structure the schema.md to include:
1. **Overview** — what each set JSON file represents
2. **Top-level fields** — schemaVersion (int, required), id (str, kebab-case), name (str),
   description (str), pieceCount (int), steps (array)
3. **Step fields** — stepNumber (int, 1-indexed, sequential), description (str), pieces (array)
4. **Piece fields** — id (str, unique within set), type (one of the 13 listed types), color
   (CSS hex), gridX (int), gridZ (int), layer (int, 0 = on baseplate), rotation (int, 0/90/180/270)
5. **Valid piece types** — list all 13 v1 types:
   brick-1x1, brick-1x2, brick-1x3, brick-1x4, brick-2x2, brick-2x3, brick-2x4,
   plate-1x1, plate-1x2, plate-1x4, plate-2x2, plate-2x4, slope-2x1, slope-2x2
6. **Coordinate system** — explain integer stud grid: gridX is column (positive = right),
   gridZ is row (positive = away from viewer), layer is stack height (0 = baseplate surface,
   each brick layer adds 1, each plate layer adds 1 — layers are in plate-heights for plates,
   brick-heights for bricks). Document that world-space conversion is: X = gridX * 8,
   Z = gridZ * 8, Y = layer * 9.6 (bricks) or layer * 3.2 (plates)
7. **Schema versioning** — schemaVersion starts at 1; bump on any breaking change to field names or types
8. **Example** — a minimal 2-step, 3-piece set example using the actual JSON structure

Do NOT use floating-point positions in the example — integer grid coords only (pitfall 2 prevention).

**File: app.py**

Structure:
```python
import os, json, glob
from flask import Flask, jsonify, send_from_directory, abort
from flask_cors import CORS
from dotenv import load_dotenv

load_dotenv()

app = Flask(__name__, static_folder='static', static_url_path='')
CORS(app)

SETS_DIR = os.getenv('SETS_DIR', 'sets')

# ---- Schema validation ----

VALID_TYPES = {
    'brick-1x1', 'brick-1x2', 'brick-1x3', 'brick-1x4',
    'brick-2x2', 'brick-2x3', 'brick-2x4',
    'plate-1x1', 'plate-1x2', 'plate-1x4', 'plate-2x2', 'plate-2x4',
    'slope-2x1', 'slope-2x2',
}
VALID_ROTATIONS = {0, 90, 180, 270}


def validate_set(data: dict, filepath: str) -> list[str]:
    """Return list of validation errors. Empty list = valid."""
    errors = []
    for field in ('schemaVersion', 'id', 'name', 'description', 'pieceCount', 'steps'):
        if field not in data:
            errors.append(f"Missing top-level field: {field}")
    if 'steps' in data:
        for si, step in enumerate(data['steps']):
            for sf in ('stepNumber', 'description', 'pieces'):
                if sf not in step:
                    errors.append(f"Step {si}: missing field {sf}")
            for pi, piece in enumerate(step.get('pieces', [])):
                for pf in ('id', 'type', 'color', 'gridX', 'gridZ', 'layer', 'rotation'):
                    if pf not in piece:
                        errors.append(f"Step {si} piece {pi}: missing field {pf}")
                if piece.get('type') not in VALID_TYPES:
                    errors.append(f"Step {si} piece {pi}: invalid type '{piece.get('type')}'")
                if piece.get('rotation') not in VALID_ROTATIONS:
                    errors.append(f"Step {si} piece {pi}: rotation must be 0, 90, 180, or 270")
                for int_field in ('gridX', 'gridZ', 'layer'):
                    if int_field in piece and not isinstance(piece[int_field], int):
                        errors.append(f"Step {si} piece {pi}: {int_field} must be integer")
    return errors


# ---- Load sets at startup ----

_sets_cache: dict[str, dict] = {}

def _load_sets():
    pattern = os.path.join(SETS_DIR, '*.json')
    for filepath in glob.glob(pattern):
        with open(filepath, 'r', encoding='utf-8') as f:
            data = json.load(f)
        errors = validate_set(data, filepath)
        if errors:
            raise SystemExit(
                f"Schema validation failed for {filepath}:\n" +
                "\n".join(f"  - {e}" for e in errors)
            )
        _sets_cache[data['id']] = data

_load_sets()


# ---- Routes ----

@app.route('/')
def index():
    return send_from_directory(app.static_folder, 'index.html')

@app.route('/api/sets')
def list_sets():
    catalogue = [
        {
            'id': s['id'],
            'name': s['name'],
            'description': s['description'],
            'pieceCount': s['pieceCount'],
        }
        for s in _sets_cache.values()
    ]
    return jsonify(catalogue)

@app.route('/api/sets/<set_id>')
def get_set(set_id):
    if set_id not in _sets_cache:
        return jsonify({'error': 'Set not found'}), 404
    return jsonify(_sets_cache[set_id])


if __name__ == '__main__':
    port = int(os.getenv('FLASK_RUN_PORT', 5000))
    debug = os.getenv('FLASK_DEBUG', '0') == '1'
    app.run(port=port, debug=debug)
```

Key rules enforced:
- Always returns `jsonify()` — never raw dict (per integration gotchas in PITFALLS.md)
- flask-cors is imported and applied — headers set for dev workflow where Vite runs on :5173
- Schema validation runs at startup — fails loudly with specific field-level errors
- Integer fields validated as `isinstance(x, int)` — catches float drift at the data level
- No debug mode unless `FLASK_DEBUG=1` in env

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → app.py | Set JSON files are read from disk at startup; malformed files could crash the server |
| HTTP client → /api/sets/:id | URL parameter is user-controlled; must not allow path traversal or expose internal errors |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | sets/*.json | mitigate | Schema validation at startup rejects malformed files before they're served; validate type enum and integer constraints |
| T-01-02 | Information Disclosure | GET /api/sets/&lt;set_id&gt; | mitigate | Return `{"error": "Set not found"}` with 404 for unknown IDs; never expose file paths, stack traces, or directory listings in error responses |
| T-01-03 | Elevation of Privilege | app.py debug mode | mitigate | FLASK_DEBUG defaults to 0; debug=True exposes Werkzeug interactive debugger (code execution); only enabled via explicit env var in dev |
| T-01-04 | Information Disclosure | SystemExit on validation failure | accept | Validation errors print to server console (not HTTP response); acceptable for a personal project with no remote attackers |
| T-01-05 | Denial of Service | Large set JSON files | accept | Personal project, no public exposure; file size is self-controlled; no rate limiting needed |
</threat_model>
