1. **`initSelection(onSetSelected)`** — Called once from main.js. Stores callback. Fetches `/api/sets`, renders set cards into `#set-list`. Each card:
   - Create div with class `set-card`, containing:
     - `<canvas>` with class `set-card-thumb` (120x90) for 3D thumbnail
     - Info div with `.set-card-info` containing: name (`.set-card-name`), description (`.set-card-desc`), meta row (`.set-card-meta`) with piece count and star rating
   - **Star rating (D-03):** Derive from pieceCount — 1 star if pieceCount <= 25, 2 stars if <= 60, 3 stars if > 60. Display using repeated `\u2605` character (filled star) in `.set-card-stars` span.

   - **3D thumbnail rendering (D-02):** After appending the card to DOM, call `_renderThumbnail(fullSetDataForThumb, canvasEl)`.

2. **`_renderThumbnail(setData, canvasEl)`** (private) — Renders the completed model as a thumbnail:
   - Fetch full set data first: `const res = await fetch('/api/sets/' + setId); const setData = await res.json();`
   - Create `THREE.WebGLRenderer({ canvas: canvasEl, antialias: true, alpha: true })`, setSize(120, 90)
   - Create `THREE.Scene()` with background `0x1a1a2e` (matches project dark theme)
   - Add ambient light (intensity 0.6) + directional light (intensity 1.0, position 40,60,50)
   - Create `THREE.PerspectiveCamera(45, 120/90, 0.1, 500)`
   - Iterate ALL steps, ALL pieces — for each piece call `getGeometry(piece.type)` and create a mesh with `MeshStandardMaterial({ color: piece.color, roughness: 0.6, metalness: 0.1 })`. Position via `gridToWorld(piece.gridX, piece.gridZ, piece.layer, piece.type)`. Apply rotation: `mesh.rotation.y = THREE.MathUtils.degToRad(piece.rotation || 0)`.
   - Compute bounding box of all meshes. Position camera to frame the model: use box center as lookAt target, distance = max dimension / Math.tan(FOV/2) * 1.5
   - Call `renderer.render(scene, camera)` once
   - Store `canvasEl.toDataURL()` as img src, replace canvas with `<img>` element to free WebGL context
   - Call `renderer.dispose()` and dispose all materials
   - Process thumbnails sequentially (one renderer at a time) to avoid browser WebGL context limit

function _cleanupBuild() {
